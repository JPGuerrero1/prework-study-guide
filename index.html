HTML

Note: HTML elements must be encapsulated in angle brackets (<>).


HTML Elements


  • The 'head' element contains information, or metadata, about the webpage and is not visible to the user.

  • The 'body' element contains all of the visible content we want shown to the user.

  • The 'link' element connects an external source to the HTML file, and is followed by the 'rel' attribute, which specifies the relation between the current and linked file.

  • The 'meta' element defines the metadata in an HTML file like the character set, description, keywords, author, and the viewport.

HTML Body Elements


  • The elements 'h1' through 'h6' create headings that will be visible in the web application. The higher number the heading, the smaller the font unless stated otherwise in an external CSS file.

  • The 'p' element creates a paragraph of text.

  • The 'ul' and 'ol' elements stand for 'unordered list' (bullet points) and 'ordered list' (numbered or lettered). To put listed items in these lists, use the 'li' element.

  • The 'br' element creates a text break.

  • The 'section' element allows a body of code and text to be separated into identifiable parts with thematic content.

CSS

Note: CSS declarations have a property followed by a value, and ended with a semicolon (;).


CSS Styles


  • Inline CSS directly styles an element in the HTML file, it applies to a specific tag using the 'style' HTML attribute.

  • Internal CSS contains rules for a webpage in the head section of an HTML file, these rules only apply to the page, but can style multiple elements using the same 'style' attribute.

  • External CSS allows rules for a webpage to be kept in a different file separate from the initial HTML file. It can be added using the 'link' element, 'ref' and 'href' attributes, and the file location of the style sheet.

CSS Capabilities


  • Visual aspects of a webpage that can be styled with CSS are text scaling, image scaling, and text alignment.

  • For a more in depth styling, CSS can also change aspects like underlining, bolding, italicising, shadows, and side-margin spacing.

Git


Git Navigation


  • The navigation commands 'cd' and 'cd ..' allow you to navigate into and out of directories on your local machine.

  • The navigation commands 'ls' and 'pwd' show you what directories are available from your current one, and what directory you're currently in.

  • The navigation commands 'mkdir' and 'touch', followed by a unique name, allow you to make a new directory, and add a file into a directory.


Git Commands


  • The git command 'git branch' creates a list of available branches on your local. Adding '-r' or '-a', will give you a similar result, but instead, '-r' showing your remote branches, and '-a' showing both your local and remote branches.

  • The git command 'git branch -d' will delete a branch on your local repo. Adding '-r', followed by the remote name, forward slash (/), and the branch to be deleted, will remove it from your local and remote tracking repos.

  • The git command 'git checkout', followed by an existing branch, will switch you from your current branch, to the one specified in the command.

  • The git command 'git checkout -b' followed by a unique branch name will create a new branch off of the main and will switch you over to the new working branch you've created.

  • The git commands 'git pull' and 'git push', followed by the remote name and the branch you wish to pull or push, will move data from your remote to your local repo (pull), and from your local to your remote repo (push).


Sending a commit in Git


  • Start by prepping your file(s) using the 'git add -A', putting your file(s) into a staging phase.

  • Then, use the 'git commit -m' command, followed by a short description of the changes you've made enclosed in quotation marks (""), to commit your changes locally.

  • Following this, use the 'git pull origin main' command to make sure your local main branch is up to date with the remote main branch; this is to ensure you are not working with outdated code.

  • Finally, use the 'git push' command, followed by the remote name and the branch you wish to push, to push your changes to the remote repo on Github, and prep for a pull request.


JavaScript


JavaScript Keywords


  • The keyword 'var' is used to create a variable, and is followed by a unique name for the variable, an equal sign (=), and an assignment operator. This is the data that will be assigned to the variable.

  • The keyword 'function' is a reusable block of code that performs a specific task, similar to 'var' in the sense that we can use it to declare a new variable with a unique name. However, with a function we can also define parameters inside parenthesis so that when we call the function later, it can execute the given parameters.

  • The keyword 'console.log' outputs a message to the application's web console using an argument. An argument is typically a name previously assigned to a variable or a function.

  • The keyword 'for' is used to create loops for code that will be continuosly executed until certain parameters are met; this is known as a 'for loop'.

  • The keyword 'if' is used to create conditional statements for the code to execute only if certain parameters are met.

JavaScript Tips


  • If there are multiple possibilities in a 'for loop', you can use an 'else if' statement to account for this. Meaning, you can have multiple different outputs depending on the input.
    (i.e. 'if' path = blocked, action = go around it. 'Else if' path = clear, action = continue walking path).

  • An 'array' is a single variable used to store multiple kinds of similar data. This is similar to a variable, however instead of assigning one value, you can assign multiple values by grouping them together with a set of brackets ([]), and separating the values with commas (,). Note that an array of data is numbered starting from '0'.
